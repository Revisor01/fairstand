**Problem:** Die settings-Tabelle hat `key: text('key').primaryKey()` — zwei Shops mit dem Key `'admin_email'` würden sich beim Upsert gegenseitig überschreiben.

**Schritt 1 — schema.ts:**

Ändere den `settings`-Import oben: füge `uniqueIndex` zur Importliste hinzu:
```typescript
import { pgTable, text, integer, boolean, serial, jsonb, uniqueIndex } from 'drizzle-orm/pg-core';
```

Ändere die settings-Tabellendefinition:
```typescript
export const settings = pgTable('settings', {
  key: text('key').notNull(),          // nicht mehr primaryKey()
  value: text('value').notNull(),
  shopId: text('shop_id').notNull(),
}, (table) => ({
  pk: uniqueIndex('settings_key_shop_id_idx').on(table.key, table.shopId),
}));
```

Der Primary Key wechselt von `key` allein zu einem Composite Unique Index auf `(key, shopId)`. Das erlaubt verschiedenen Shops, denselben Setting-Key zu verwenden.

**Schritt 2 — settings.ts:**

Ändere den `onConflictDoUpdate`-Target auf das Composite-Unique:
```typescript
await db.insert(settings)
  .values({ key: s.key, value: s.value, shopId: session.shopId })
  .onConflictDoUpdate({
    target: [settings.key, settings.shopId],   // ← Composite target
    set: { value: s.value },
  });
```
