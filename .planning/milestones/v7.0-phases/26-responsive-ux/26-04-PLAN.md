Ändere ArticleGrid.tsx an folgenden Stellen:

1. **Import ergänzen** — `useRef` und `useEffect` aus React importieren (falls noch nicht vorhanden):
```tsx
import { useMemo, useState, useRef, useEffect } from 'react';
```

2. **Refs für Auto-Scroll** — nach den useMemo-Aufrufen:
```tsx
const tabsContainerRef = useRef<HTMLDivElement>(null);
const activeTabRef = useRef<HTMLButtonElement>(null);
```

3. **useEffect für Auto-Scroll** — nach den Ref-Deklarationen:
```tsx
useEffect(() => {
  activeTabRef.current?.scrollIntoView({
    behavior: 'smooth',
    block: 'nearest',
    inline: 'center',
  });
}, [activeCategory]);
```

4. **Kategorie-Container** umbauen — den bestehenden `<div className="flex gap-1 overflow-x-auto ...">` ersetzen durch:
```tsx
<div
  ref={tabsContainerRef}
  className="
    sticky top-[68px] z-20
    flex gap-2 overflow-x-auto px-4 py-3
    bg-white border-b border-sky-100
    scrollbar-hide
    shrink-0
  "
>
```

Hinweis zu `scrollbar-hide`: Tailwind 4 mit @tailwindcss/vite unterstützt `scrollbar-hide` nativ. Falls es nicht kompiliert, als Fallback CSS einfügen: in der Klasse ergänze `[&::-webkit-scrollbar]:hidden`.

5. **Aktive Pill** — `bg-sky-400 text-white` ersetzen durch:
```
bg-sky-500 text-white shadow-md font-semibold
```

6. **Inaktive Pill** — `bg-sky-100 text-sky-700 hover:bg-sky-200` ersetzen durch:
```
bg-sky-50 text-sky-600 hover:bg-sky-100
```

7. **Ref-Zuweisung** in der Pill-Schleife — jeder button bekommt `ref` wenn aktiv:
```tsx
<button
  key={cat}
  ref={activeCategory === cat ? activeTabRef : null}
  onPointerDown={() => setActiveCategory(cat)}
  className={`
    px-4 py-2 rounded-full text-sm font-medium whitespace-nowrap min-h-[44px] transition-colors
    ${activeCategory === cat
      ? 'bg-sky-500 text-white shadow-md font-semibold'
      : 'bg-sky-50 text-sky-600 hover:bg-sky-100'
    }
  `}
>
  {cat}
</button>
```

Das äußere `<div className="flex flex-col h-full">` bleibt unverändert. Der Produkt-Grid-Bereich (`flex-1 overflow-y-auto`) bleibt unverändert.
