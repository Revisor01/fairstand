```tsx
interface CartPanelProps {
  isOpen: boolean;
  sidebar?: boolean;  // NEU: true = statische Spalte, false/undefined = Slide-In
  items: CartItem[];
  total: number;
  onClose: () => void;
  onUpdateQuantity: (productId: string, quantity: number) => Promise<{ blocked?: boolean; stock?: number }>;
  onRemoveItem: (productId: string) => void;
  onCheckout: () => void;
  onWithdrawal?: () => void;

```

3. **Render-Logik** umbauen: Wenn `sidebar === true`, wird kein `fixed`-Overlay und keine `translate-x`-Transition gerendert, sondern ein statisches `<div>`:

```tsx
// Sidebar-Modus: statische Spalte, kein Overlay
if (sidebar) {
  return (
    <div className="w-80 bg-white border-l border-sky-100 flex flex-col h-full">
      {/* Header */}
      <div className="flex items-center justify-between px-4 py-3 border-b border-sky-100 shrink-0">
        <h2 className="text-lg font-semibold text-slate-800">Warenkorb</h2>
      </div>
      {/* Stock-Warnung, Artikel-Liste und Footer — identisch zum Slide-In */}
      {/* ... copy bestehenden Panel-Inhalt (ohne X-Button, ohne Overlay) ... */}
    </div>
  );
}

// Slide-In Modus: bestehendes Verhalten (Fragment mit Overlay + fixed div)
return (
  <>
    {isOpen && (
      <div className="fixed inset-0 bg-black/30 z-40" onPointerDown={onClose} />
    )}
    <div className={`fixed inset-y-0 right-0 w-80 md:w-96 bg-white shadow-2xl z-50
      flex flex-col transform transition-transform duration-300

      {/* Header mit X-Button */}
      {/* ... bestehender Inhalt ... */}
    </div>
  </>
);
```

Wichtig: Der Panel-Inhalt (Artikel-Liste, Footer mit Bezahlen-Button) ist in BEIDEN Modi identisch. Nur das äußere Container-Element und das Overlay unterscheiden sich. Refaktoriere den gemeinsamen Inhalt in eine `<CartPanelContent>` Subkomponente oder wiederhole ihn per Copy — was sauberer ist.

Baue POSScreen.tsx um, um das responsive Sidebar-Layout zu implementieren:

1. **Neue State-Variablen** (nach isOnline State):
```tsx
const [cartSidebarEnabled, setCartSidebarEnabled] = useState(false);
const [isLargeScreen, setIsLargeScreen] = useState(() =>
  window.matchMedia('(min-width: 1024px)').matches
);
```

2. **Settings laden** — neuer useEffect nach dem shopName-useEffect:
```tsx
useEffect(() => {
  getAuthHeaders().then(headers =>
    fetch('/api/settings', { headers })
      .then(r => r.json())
      .then((rows: Array<{ key: string; value: string }>) => {
        const setting = rows.find(r => r.key === 'cart_sidebar_enabled');
        if (setting) setCartSidebarEnabled(setting.value === 'true');
      })
      .catch(() => {})
  );
}, []);
```

3. **MediaQuery reaktiv überwachen** — neuer useEffect:
```tsx
useEffect(() => {
  const mq = window.matchMedia('(min-width: 1024px)');
  const handler = (e: MediaQueryListEvent) => setIsLargeScreen(e.matches);
  mq.addEventListener('change', handler);
  return () => mq.removeEventListener('change', handler);
}, []);
```

4. **Derived State** (nach den State-Deklarationen):
```tsx
const shouldShowSidebar = cartSidebarEnabled && isLargeScreen;
```

5. **Header-Button** bedingen: Den Warenkorb-Button im Header nur zeigen wenn `!shouldShowSidebar`:
```tsx
{!shouldShowSidebar && (
  <button
    onPointerDown={() => setIsCartOpen(true)}
    className="relative min-h-[44px] min-w-[44px] flex items-center justify-center rounded-xl active:bg-sky-500 transition-colors"
    aria-label="Warenkorb öffnen"
  >
    <ShoppingCart size={26} />
    {cartItemCount > 0 && (
      <span className="absolute -top-1 -right-1 bg-white text-sky-700 text-xs font-bold w-5 h-5 rounded-full flex items-center justify-center">
        {cartItemCount}
      </span>
    )}
  </button>
)}
```

6. **Layout-Container** im POS-Hauptansicht-Block umbauen — das `<div className="min-h-screen bg-sky-50 flex flex-col">` wird zu:
```tsx
<div className={`min-h-screen bg-sky-50 flex flex-col ${shouldShowSidebar ? 'lg:flex-row' : ''}`}>
  {/* LowStockBanner + Header bleiben oben */}
  <LowStockBanner />
  <header ...>...</header>

  {/* Inhalt-Bereich: flex-row auf lg+ wenn Sidebar aktiv */}
  <div className="flex flex-1 overflow-hidden">
    {/* Artikel-Grid */}
    <div className="flex-1 overflow-hidden">
      <ArticleGrid onAddToCart={...} />
    </div>

    {/* Sidebar-Modus: statische Spalte */}
    {shouldShowSidebar && (
      <CartPanel
        sidebar={true}
        isOpen={true}
        items={cart.items}
        total={cart.total}
        onClose={() => {}}
        onUpdateQuantity={cart.updateQuantity}
        onRemoveItem={cart.removeItem}
        onCheckout={() => { setView('payment'); }}
        onWithdrawal={handleWithdrawal}
      />
    )}
  </div>

  {/* Slide-In Modus: bestehendes Verhalten */}
  {!shouldShowSidebar && (
    <CartPanel
      isOpen={isCartOpen}
      items={cart.items}
      total={cart.total}
      onClose={() => setIsCartOpen(false)}
      onUpdateQuantity={cart.updateQuantity}
      onRemoveItem={cart.removeItem}
      onCheckout={() => { setIsCartOpen(false); setView('payment'); }}
      onWithdrawal={handleWithdrawal}
    />
  )}
</div>
```

Hinweis zur onAddToCart-Logik: Im Sidebar-Modus muss `setIsCartOpen(true)` NICHT aufgerufen werden (Warenkorb ist immer sichtbar). Ändere die onAddToCart-Callback:

```tsx
onAddToCart={product => {

  if (!result.added) {
    setStockError(`"${product.name}" ist nicht mehr auf Lager.`);
    setTimeout(() => setStockError(null), 2500);
  } else {
    setStockError(null);
    if (!shouldShowSidebar) setIsCartOpen(true);  // nur im Slide-In Modus öffnen
  }
}}
```

Stockfehler-Toast und alle anderen bestehenden Funktionen bleiben unverändert.
