## Phases

### Phase 30: Admin-Verwaltung
**Goal:** Die Produktverwaltung ist vollständig — Artikel können dauerhaft gelöscht werden, die Inventur ist direkt zugänglich, und Beschriftungen sind eindeutig
**Plans:** 2 (30-01: Backend DELETE + Unique-Constraint, 30-02: Frontend InventurTab + Löschen-UI + Umbenennung)
**Completed:** 2026-04-02

Key deliverables:
- DELETE /api/products/:id with sales history check (409 if sales exist)
- Unique constraint on article_number + shop_id
- InventurTab extracted from MonthlyReport as own Admin tab
- Delete button with confirmation dialog in ProductList
- "Bestandskorrektur" → "Bestand anpassen"

### Phase 31: Tagesübersicht-UX
**Goal:** Die Tagesübersicht zeigt auf einen Blick welche Transaktionen Spenden enthielten und ermöglicht eine freie Datumsauswahl per Kalender
**Plans:** 1 (31-01: Spendenmarkierung + Datepicker)
**Completed:** 2026-04-03

Key deliverables:
- Sales with donations highlighted green (bg-green-50, bold green amount)
- Native date picker styled as active button when selected

### Phase 32: Auto-Logout
**Goal:** Ein abgelaufener Session-Token führt zu einem sauberen Redirect auf den PIN-Login — kein verwirrender 401-Fehler
**Plans:** 1 (32-01: CustomEvent auth:logout)
**Completed:** 2026-04-03

Key deliverables:
- authFetch dispatches CustomEvent('auth:logout') instead of window.location.reload()
- useAuth listens for event and sets state to 'locked'
- Clean React state transition, no white screen

## Additional Quick-Fixes (outside phase scope)

During v9.0 development, several quick-fixes were implemented:
- Live available stock in article grid (stock minus cart quantity)
- Plus button disabled at stock limit with "Max. Bestand: X" hint
- Receipt download on sale summary page (with/without donation)
- Withdrawals in inventory (Entnahme column + EK costs)
- Price period breakdown in inventory (EK/VK history per period)
- Balance section in inventory (revenue, withdrawal income, EK costs, margin)
- Total purchase cost overview in inventory balance
- Content-Type fix for bodyless DELETE/GET requests
- PWA autoUpdate instead of prompt (prevents stale cache)

## Decisions

- Artikel löschen nur wenn keine Verkäufe (Fehlerkorrektur-Mechanismus)
- Token-Expiration via CustomEvent statt Page-Reload
