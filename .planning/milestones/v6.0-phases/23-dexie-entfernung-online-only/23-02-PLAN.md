<!-- API-Kontrakt von idb-keyval (wird durch localStorage ersetzt) -->

idb-keyval API (bisherig):
```typescript
import { get, set, del } from 'idb-keyval';
await set('key', value);        // speichert beliebigen Wert
const val = await get<T>('key'); // liest Wert, undefined wenn nicht vorhanden
await del('key');               // löscht Schlüssel
```

localStorage API (neu):
```typescript
// Für Objekte/Arrays:
localStorage.setItem('key', JSON.stringify(value));
const raw = localStorage.getItem('key');
const val = raw ? JSON.parse(raw) as T : null;
localStorage.removeItem('key');

// Für primitive Strings:
localStorage.setItem('key', value);
const val = localStorage.getItem('key') ?? null;
```

WICHTIG: idb-keyval-Funktionen sind async — in den betroffenen Dateien sind die aufrufenden Funktionen alle async.
Nach Umstellung auf localStorage sind die Operationen synchron.
Die Funktions-Signaturen BLEIBEN async (await bleibt syntaktisch möglich, wird zu no-op).
Dies verhindert Änderungen in allen Aufrufern.
