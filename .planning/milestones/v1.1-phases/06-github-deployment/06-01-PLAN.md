<task type="auto">
  <name>Task 2: GitHub Repository erstellen und Code pushen</name>
  <files></files>
  <read_first>.gitignore, .github/workflows/deploy.yml</read_first>
  <action>
1. GitHub Repository erstellen:
   `gh repo create Revisor01/fairstand --private --source=. --push`

   Falls das fehlschlaegt wegen bereits existierendem Remote, dann:
   - `git remote add origin https://github.com/Revisor01/fairstand.git`
   - `git push -u origin main`

2. Pruefen ob der Push erfolgreich war:
   `gh repo view Revisor01/fairstand`

3. Pruefen ob GitHub Actions Workflow getriggert wurde:
   `gh run list --limit 1`

   Der Workflow wird zunaechst fehlschlagen beim Portainer-Webhook-Schritt (Secret fehlt noch) — das ist erwartet und wird in Plan 03 behoben.

WICHTIG: Das Repo als PRIVATE erstellen. Der GitHub-Account ist `Revisor01` (nicht simonluthe).
  </action>
  <verify>
    <automated>gh repo view Revisor01/fairstand --json name,visibility 2>/dev/null && echo "OK"</automated>
  </verify>
  <acceptance_criteria>
- `gh repo view Revisor01/fairstand` zeigt das Repository
- `gh run list --limit 1` zeigt mindestens einen Workflow Run
- `git remote -v` zeigt origin mit Revisor01/fairstand
  </acceptance_criteria>
  <done>GitHub Repository Revisor01/fairstand existiert, Code ist gepusht, GitHub Actions Workflow wurde getriggert (GHCR-Push laeuft oder ist abgeschlossen)</done>
</task>
