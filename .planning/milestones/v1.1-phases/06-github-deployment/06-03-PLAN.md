<task type="checkpoint:human-action" gate="blocking">
  <name>Task 1: Portainer Stack deployen</name>
  <files></files>
  <action>
Der Stack muss auf Portainer deployed werden. Die GHCR Images muessen vom Server aus pullbar sein.

**Schritt 1: GHCR Registry in Portainer konfigurieren (falls noch nicht vorhanden)**
1. Oeffne https://docker.godsapp.de
2. Gehe zu Settings → Registries → Add registry
3. Waehle "Custom registry"
4. Name: `GHCR`
5. Registry URL: `ghcr.io`
6. Authentication: Username = `Revisor01`, Password = ein GitHub Personal Access Token mit `read:packages` Scope
   (Erstelle unter https://github.com/settings/tokens/new mit Scope `read:packages`)

**Schritt 2: Stack erstellen**
1. In Portainer: Stacks → Add stack
2. Name: `fairstand`
3. Build method: "Web editor"
4. Kopiere den Inhalt von `docker-compose.portainer.yml` in den Editor
5. Unter "Environment variables" die optionalen SMTP-Variablen eintragen (falls gewuenscht):
   - SMTP_HOST, SMTP_PORT, SMTP_SECURE, SMTP_USER, SMTP_PASS, SMTP_FROM
   (Koennen auch spaeter nachgetragen werden)
6. Klicke "Deploy the stack"

**Schritt 3: Pruefen**
- Beide Container (client + server) muessen "running" sein
- fairstand.godsapp.de sollte jetzt die App anzeigen

HINWEIS: Die GHCR-Images muessen von Plan 01 (GitHub Actions) erfolgreich gebaut worden sein.
  </action>

<task type="auto">
  <name>Task 2: Portainer Webhook erstellen und GitHub Secret setzen</name>
  <files></files>
  <read_first>.github/workflows/deploy.yml</read_first>
  <action>
**Schritt 1: Webhook-URL vom User erfragen**
Der User muss in Portainer den Webhook erstellen:
1. In Portainer: Stacks → fairstand → Stack settings (oder direkt im Stack-Detail)
2. "Create a webhook" aktivieren / Webhook erstellen
3. Die generierte Webhook-URL kopieren (Format: https://docker.godsapp.de/api/stacks/webhooks/XXXXXX)

**Schritt 2: GitHub Secret setzen (kann Claude automatisieren)**
Sobald die Webhook-URL bekannt ist:
```bash
gh secret set PORTAINER_WEBHOOK_URL --repo Revisor01/fairstand --body "WEBHOOK_URL_HIER"
```

**Schritt 3: Deployment testen**
Einen leeren Commit pushen um den vollen Cycle zu testen:
```bash
git commit --allow-empty -m "ci: test deployment pipeline"
git push origin main
```

Dann pruefen:
```bash
gh run list --limit 1
gh run view --log-failed $(gh run list --limit 1 --json databaseId -q '.[0].databaseId') 2>/dev/null || echo "Run erfolgreich oder noch laufend"
```
