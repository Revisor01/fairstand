**Server-Sync nach Dexie-Write (per Warning: Produkt-CRUD muss Server-Sync haben):**
Nach erfolgreichem Dexie-Write, wenn online (`navigator.onLine`), fire-and-forget fetch an den Server:

```typescript
const productData = { id, shopId: SHOP_ID, articleNumber, name, category, purchasePrice, salePrice, vatRate, stock, minStock, active: true, updatedAt: Date.now() };
await db.products.add(productData); // oder .update()

// Server-Sync bei Online-Status (fire-and-forget, Fehler loggen aber nicht blockieren)
if (navigator.onLine) {
  fetch('/api/products', {
    method: 'POST',
    headers: { 'Content-Type': 'application/json' },
    body: JSON.stringify(productData),
  }).catch(err => console.warn('Server-Sync fehlgeschlagen:', err));
}
```

Der POST-Endpoint auf dem Server ist bereits ein LWW-Upsert, daher funktioniert sowohl Anlegen als auch Bearbeiten ueber denselben Endpoint.

Nach dem Speichern: `onClose()` aufrufen.

Validation: Name und Artikelnummer muessen ausgefuellt sein, Preise >= 0.

Styling: Modal/Overlay oder eigene Seite im Admin-Tab. Formular-Inputs mit `h-12 text-lg` fuer Touch.

**1. client/src/hooks/useLowStockCount.ts (NEU):**

Wiederverwendbarer Hook der die Anzahl der Produkte mit Mindestbestand-Unterschreitung liefert. Wird sowohl vom LowStockBanner als auch vom Badge im Admin-Button genutzt (per CONTEXT.md locked decision: "Badge im Admin-Bereich + Banner auf dem Kassen-Screen").

```typescript
import { useLiveQuery } from 'dexie-react-hooks';
import { db, SHOP_ID } from '../db/index.js';

export function useLowStockProducts() {
  return useLiveQuery(
    () => db.products
      .where('[shopId+active]')
      .equals([SHOP_ID, 1])
      .filter(p => p.minStock > 0 && p.stock <= p.minStock)
      .toArray(),
    [],
    []
  );
}

export function useLowStockCount(): number {
  const products = useLowStockProducts();
  return products.length;
}
```

import { useLowStockProducts } from '../../hooks/useLowStockCount.js';

export function LowStockBanner() {
