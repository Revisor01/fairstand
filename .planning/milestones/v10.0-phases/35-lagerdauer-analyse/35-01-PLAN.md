In server/src/routes/products.ts den GET /products Endpoint (aktuell: einfaches `db.select().from(products)`) durch eine SQL-Abfrage ersetzen, die für jedes Produkt das `last_sale_at` berechnet:

```sql
SELECT
  p.*,
  MAX(CASE WHEN s.cancelled_at IS NULL THEN s.created_at ELSE NULL END) AS last_sale_at
FROM products p
LEFT JOIN LATERAL (
  SELECT s2.created_at, s2.cancelled_at
  FROM sales s2
  WHERE s2.shop_id = p.shop_id
    AND s2.items::jsonb @> json_build_array(json_build_object('productId', p.id))::jsonb
) s ON true
WHERE p.shop_id = ${shopId}
GROUP BY p.id
```

Da der LATERAL JOIN mit Drizzle-ORM komplex ist, direkt `db.execute(sql\`...\`)` verwenden (wie in sales.ts bereits gemacht). Das Ergebnis als JSON-Array zurückgeben.

Wichtig: `last_sale_at` ist NULL wenn der Artikel noch nie verkauft wurde, oder wenn alle Verkäufe storniert waren. Der Wert ist ein Unix-Timestamp in Millisekunden (bigint in PostgreSQL → als Number in JSON).

Der bestehende Import von `sql, eq, and` aus drizzle-orm bleibt erhalten. `db` Import bleibt unverändert.

Ladenhüter-Zähler analog zu `outOfStockCount`:
```typescript
const ladenhüterCount = useMemo(() => {
  if (!products) return 0;
  return products.filter(p => {

    return days === null || days > 90;
  }).length;
}, [products]);
```
