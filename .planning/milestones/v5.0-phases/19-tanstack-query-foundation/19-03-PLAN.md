```typescript
// 'offlineFirst': Query läuft immer, Cache wird bevorzugt; Network-Request im Hintergrund

```

    ArticleGrid.tsx migrieren:

    1. Imports anpassen:
    ```tsx
    // ENTFERNEN:
    import { useLiveQuery } from 'dexie-react-hooks';

    // HINZUFÜGEN:
    ```

    HINWEIS: `useProducts()` aus den Hooks kann für den POS nicht direkt wiederverwendet werden, weil die networkMode-Einstellung pro Query-Call gesetzt werden muss. Daher useQuery direkt in ArticleGrid aufrufen (nicht den useProducts-Hook), um 'offlineFirst' zu setzen:

    ```tsx
    const { data: products, isLoading } = useQuery<Product[]>({
      queryKey: ['products', getShopId()],
      queryFn: async () => {
        const headers = await getAuthHeaders();
        const shopId = getShopId();
        const res = await fetch(`/api/products?shopId=${shopId}`, { headers });
        if (!res.ok) throw new Error('Produkte konnten nicht geladen werden');
        const data = await res.json() as Record<string, unknown>[];
        // Server liefert snake_case — gleiche Mapping-Logik wie in useProducts.ts
        return data.map(p => ({
          id: p.id as string,
          shopId: (p.shop_id ?? p.shopId) as string,
          articleNumber: (p.article_number ?? p.articleNumber) as string,
          name: p.name as string,
          category: (p.category ?? '') as string,
          purchasePrice: (p.purchase_price ?? p.purchasePrice ?? 0) as number,
          salePrice: (p.sale_price ?? p.salePrice) as number,
          vatRate: (p.vat_rate ?? p.vatRate ?? 7) as number,
          stock: (p.stock ?? 0) as number,
          minStock: (p.min_stock ?? p.minStock ?? 0) as number,
          active: (p.active ?? true) as boolean,
          updatedAt: (p.updated_at ?? p.updatedAt ?? Date.now()) as number,
          imageUrl: (p.image_url ?? p.imageUrl) as string | undefined,
        })).filter(p => p.active); // Nur aktive Produkte für POS
      },
      networkMode: 'offlineFirst',
      staleTime: 60_000, // POS: 1 Minute — etwas länger als Admin, da POS stabiler ist
    });
    ```

    WARUM queryFn direkt statt useProducts(): Der useProducts-Hook in hooks/api/useProducts.ts hat keinen networkMode-Parameter — die Einstellung 'offlineFirst' ist POS-spezifisch und gehört in den Aufrufer. Durch gleichen queryKey ['products', shopId] teilen sich Admin und POS den gleichen Cache — ein Admin-Update invalidiert auch den POS-Cache.

    2. Filterung auf active-Produkte: Im bisherigen Code wird `p.active === true` als Dexie-Filter gesetzt. Das gleiche passiert jetzt per `.filter(p => p.active)` nach dem Mapping. KEIN separater useMemo nötig — ist inline im queryFn.

    3. Ladestate:
    ```tsx
    // ALT: if (!products)
    // NEU: if (isLoading)
    // Beide führen zum gleichen Render-Zweig
    ```

    4. Kategorie-Filter-Logik (categories useMemo, filteredProducts useMemo) bleibt exakt unverändert — arbeitet einfach auf `products` statt `useLiveQuery`-Ergebnis.

    5. Die Kategorie-Tabs und das Produkt-Grid JSX bleiben vollständig unverändert.

    WICHTIG: `db` Import entfernen — ArticleGrid braucht keine direkte Dexie-Verbindung mehr.
    `getShopId` Import bleibt (wird für queryKey benötigt).
