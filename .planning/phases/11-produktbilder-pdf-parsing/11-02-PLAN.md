  <action>
    In `server/src/routes/products.ts` zwei neue Endpunkte hinzufügen. `@fastify/multipart` ist bereits als Dependency vorhanden und in `importRoutes` genutzt — hier aber direkt im Route-Handler registrieren (gleiche Strategie wie importRoutes: intern registrieren).

    ACHTUNG: `importRoutes` registriert `@fastify/multipart` bereits global auf der Fastify-Instanz — daher in `productRoutes` NICHT nochmals `fastify.register(multipart, ...)` aufrufen, sondern `request.file()` direkt nutzen. Falls ein Konflikt auftritt, als separate Fastify-Plugin-Instanz kapseln.

    **POST /api/products/:id/image**
    ```
    - Nimmt multipart/form-data mit Feld "image" entgegen
    - Akzeptiert nur image/jpeg, image/png, image/webp (MIME-Check)
    - Datei max. 5 MB (limits bereits via multipart gesetzt)
    - Speichert als /app/data/images/{productId}.{ext} (ext aus MIME ableiten)
    - Schreibt mit fs/promises writeFile (Node 20, kein Import-Overhead)
    - Aktualisiert imageUrl in SQLite: UPDATE products SET image_url = '/api/images/{filename}' WHERE id = :id
    - Gibt { ok: true, imageUrl: '/api/images/{filename}' } zurück
    - Fehler 400 wenn kein Bild, 415 wenn falscher MIME-Typ
    ```

    **GET /api/images/:filename**
    ```
    - Liest /app/data/images/:filename mit fs/promises readFile
    - Setzt Content-Type korrekt (jpeg/png/webp)
    - 404 wenn Datei nicht existiert
    - Kein @fastify/static nötig — manuelle Route reicht für diese Menge an Dateien
    ```

    **Verzeichnis anlegen:** Am Anfang von `productRoutes`:
    ```typescript
    import { mkdir } from 'node:fs/promises';
    const IMAGES_DIR = process.env.IMAGES_DIR ?? '/app/data/images';
    // Beim Route-Init:
    await mkdir(IMAGES_DIR, { recursive: true });
    ```

    Die `imageUrl`-Felder in GET /products und POST /products sind bereits in Plan 11-01 erledigt — hier nur die zwei neuen Endpunkte ergänzen.

    c) `handleImageUpload`-Funktion im Component:
    ```typescript
    async function handleImageUpload(productId: string, file: File | undefined) {
      if (!file || !navigator.onLine) return;
      const formData = new FormData();
      formData.append('image', file);
      try {
        const res = await fetch(`/api/products/${productId}/image`, {
          method: 'POST',
          body: formData,
        });
        if (!res.ok) return;
        const { imageUrl } = await res.json() as { imageUrl: string };
        await db.products.update(productId, { imageUrl });
      } catch {
        // Stilles Fail — kein Offline-Support für Bilder nötig
      }
    }
    ```
