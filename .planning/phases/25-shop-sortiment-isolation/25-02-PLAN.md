Der Import-Screen übergibt shopId bereits korrekt an neue Produkte (getShopId()), legt aber den Produkt-Fetch mit `?shopId=...` in der URL ab, was inkonsistent mit dem Session-Pattern ist.

Output: sync.ts mit shopId-Validierung vor jedem Stock-Delta; ImportScreen.tsx ohne überflüssigen shopId Query-Param im Produkt-Fetch.

**SALE_COMPLETE — Stock-Loop (Zeilen 113-117):**

Ersetze die for-Schleife durch eine mit Ownership-Prüfung:
```typescript
for (const item of sale.items) {
  const [prod] = await tx.select().from(products).where(eq(products.id, item.productId)).limit(1);
  if (!prod || prod.shopId !== entry.shopId) {
    // Produkt gehört nicht zu diesem Shop — Stock-Delta überspringen (kein Cross-Shop-Update)
    continue;
  }
  await tx.update(products)
    .set({ stock: sql`${products.stock} - ${item.quantity}` })
    .where(eq(products.id, item.productId));
}
```

Hinweis: `entry.shopId` wird bereits gegen `session.shopId` validiert (Zeilen 71-76 in syncRoutes), daher ist `entry.shopId === session.shopId` garantiert. Die Prüfung `prod.shopId !== entry.shopId` schützt vor manipulierten productIds.
