**StockAdjustModal.tsx — Toggle + EK-Eingabe hinzufügen:**

Neuer State:
```typescript
const [overridePurchasePrice, setOverridePurchasePrice] = useState(false);
const [purchasePriceInput, setPurchasePriceInput] = useState<string>(
  (product.purchasePrice / 100).toFixed(2)
);
```

Logik:
- `isPositiveDelta`: `parsedDelta > 0` — steuert Toggle-Sichtbarkeit
- Toggle "Preis anpassen" (Switch-ähnlich mit `<input type="checkbox">`) nur rendern wenn `isPositiveDelta === true`
- Wenn Toggle aus: EK-Feld ausblenden
- Wenn Toggle an: EK-Eingabe anzeigen (type="number", step="0.01", min="0"), Startwert = `(product.purchasePrice / 100).toFixed(2)`
- Bei `handleSave`: `purchasePriceCents` berechnen:
  - Wenn `isPositiveDelta && overridePurchasePrice`: `Math.round(parseFloat(purchasePriceInput) * 100)` (validieren dass > 0)
  - Wenn `isPositiveDelta && !overridePurchasePrice`: `undefined` senden (Server nutzt Fallback auf Produkt-EK per Task 1)
  - Wenn `!isPositiveDelta`: `undefined` (Server setzt null)

UI-Struktur für den Toggle-Block (nach dem Vorschau-Block, vor dem Grund-Feld):
```jsx
{isPositiveDelta && (
  <div className="flex flex-col gap-2">
    <label className="flex items-center gap-3 cursor-pointer">
      <input
        type="checkbox"
        checked={overridePurchasePrice}
        onChange={e => setOverridePurchasePrice(e.target.checked)}
        className="h-5 w-5 rounded border-slate-300 text-sky-600"
      />
      <span className="text-sm font-medium text-slate-600">Preis anpassen</span>
    </label>
    {overridePurchasePrice && (
      <div className="flex flex-col gap-1">
        <label className="text-sm font-medium text-slate-600">
          EK-Preis (€)
        </label>
        <input
          type="number"
          step="0.01"
          min="0"
          value={purchasePriceInput}
          onChange={e => setPurchasePriceInput(e.target.value)}
          className="h-12 text-lg border border-slate-200 rounded-lg px-3 focus:outline-none focus:border-sky-400"
          placeholder="z.B. 1.49"
        />
      </div>
    )}
  </div>
)}
```

WICHTIG: Bei positivem Delta Reset-Verhalten sicherstellen — wenn `parsedDelta` von positiv auf negativ wechselt, `setOverridePurchasePrice(false)` aufrufen (kein separater useEffect nötig, einfach in der handleSave-Validierung abfangen).
