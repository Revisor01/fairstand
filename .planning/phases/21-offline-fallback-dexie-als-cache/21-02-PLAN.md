flushOutbox() erhält einen optionalen `queryClient`-Parameter. Nach erfolgreichem Flush (nach `db.outbox.bulkDelete(ids)`) wird der Produkt-Cache invalidiert damit TQ frische Daten vom Server lädt.

**Import hinzufügen:**
```typescript
import type { QueryClient } from '@tanstack/react-query';
```

**Signatur ändern:**
```typescript
export async function flushOutbox(queryClient?: QueryClient): Promise<void> {
```

**Nach dem bulkDelete-Block einfügen** (vor dem finally):
```typescript
// TQ-Cache invalidieren: frische Produktdaten nach Sync
if (queryClient) {
  try {
    const shopId = getShopId();
    queryClient.invalidateQueries({ queryKey: ['products', shopId] });
  } catch {
    // getShopId() kann werfen wenn Session abgelaufen — ignorieren
  }
}
```

Der optionale Parameter stellt Rückwärtskompatibilität mit bestehenden Aufrufen in triggers.ts (ohne Argument) sicher. triggers.ts muss NICHT geändert werden.

Dann muss der `queryClient` von einem Aufrufer mit React-Kontext übergeben werden. Der richtige Aufrufer ist `useWebSocket` in `UnlockedApp` oder ein neues `useFlushWithInvalidation`-Muster. Da `useWebSocket` bereits `useQueryClient()` verwendet, kann der Flush dort erweitert werden.

Prüfen ob useWebSocket.ts oder die online/visibilitychange-Trigger in der App aktuell den queryClient übergeben können. Die Trigger in `triggers.ts` laufen außerhalb React — sie rufen `flushOutbox()` ohne queryClient auf (das ist OK, der Cache wird dann nicht invalidiert). Der Invalidation-Pfad greift nur wenn explizit übergeben.

Für OFFL-02 ist der wichtigste Pfad: nach online-Event den queryClient übergeben. Dafür die `registerSyncTriggers`-Funktion so erweitern dass sie optional einen queryClient akzeptiert:

```typescript
// triggers.ts
import type { QueryClient } from '@tanstack/react-query';
import { flushOutbox } from './engine.js';

export function registerSyncTriggers(queryClient?: QueryClient): void {
  if (navigator.onLine) {
    void flushOutbox(queryClient);
  }
  window.addEventListener('online', () => {
    void flushOutbox(queryClient);
  });
  document.addEventListener('visibilitychange', () => {
    if (document.visibilityState === 'visible' && navigator.onLine) {
      void flushOutbox(queryClient);
    }
  });
  setInterval(() => {
    if (navigator.onLine) void flushOutbox(queryClient);
  }, 30_000);
}
```

`registerSyncTriggers` wird in main.tsx ohne Argument aufgerufen — das bleibt gültig (optionaler Parameter). Der queryClient-Aufruf erfolgt dann aus einem React-Kontext (z.B. App.tsx oder UnlockedApp).

Dateiänderungen in dieser Task:
- `client/src/sync/engine.ts`: Signatur + QueryClient-Import + Invalidation nach bulkDelete
- `client/src/sync/triggers.ts`: optionaler queryClient-Parameter durchreichen

Einen kleinen Offline-Indicator im POS-Header anzeigen. Verwendet `navigator.onLine` als initialen Wert und reagiert auf `online`/`offline` Events.

**State + Effect hinzufügen** (nach den bestehenden useState-Zeilen):
```typescript
const [isOnline, setIsOnline] = useState(navigator.onLine);

useEffect(() => {
  const handleOnline = () => setIsOnline(true);
  const handleOffline = () => setIsOnline(false);
  window.addEventListener('online', handleOnline);
  window.addEventListener('offline', handleOffline);
  return () => {
    window.removeEventListener('online', handleOnline);
    window.removeEventListener('offline', handleOffline);
  };
}, []);
```
