PaymentFlow.tsx komplett neu schreiben. Die Datei ersetzt den alten 2-Step-Flow durch einen einzelnen kompakten Screen im SaleSummary-Stil.

**Neue Props-Signatur:**
```typescript
interface PaymentFlowProps {
  totalCents: number;
  items: CartItem[];           // NEU — wird aus useCart().items übergeben

  onCancel: () => void;
}
```
Import CartItem aus `../../db/index.js`.

**Layout (per Entscheidung D-01/UI-01+02 — kein fullscreen mehr):**
- Äußerer Wrapper: `flex flex-col items-center justify-center min-h-screen px-4 py-8 bg-sky-50`
- Innere Karte: `w-full max-w-sm bg-white rounded-2xl shadow-sm p-6 space-y-5`

**Aufbau der Karte (von oben nach unten):**

1. **Kopfzeile** — Titel "Bezahlung" + `onCancel`-Button (X-Icon, `onPointerDown`) oben rechts

2. **Artikelliste** (scrollbar wenn viele Artikel) — `max-h-40 overflow-y-auto`:
   - Jede Zeile: Name links, `Menge × Preis` rechts (text-sm text-slate-600)
   - Trennlinie nach der Liste

3. **Gesamtsumme** — Label "Gesamt" + Betrag `text-2xl font-bold text-slate-800` (prominent)
   - Trennlinie

4. **Bezahlt-Eingabe** — Label "Bezahlt" + `NumPad`-ähnliche Inline-Zifferneingabe:
   - State `inputStr: string` (z.B. "5,30") — Eingabe als String, keine separate NumPad-Vollseite
   - Verwende den bestehenden `<NumPad>`-Import NICHT als Vollseite — stattdessen: Ein großes `<input type="text" inputMode="decimal">` (h-14 text-xl text-center border-2 border-sky-200 rounded-xl focus:border-sky-400) plus darunter 3×3 Ziffernbuttons + "00" + "," + Backspace als Grid (ähnlich NumPad aber inline in der Karte)
   - Cent-Berechnung: `parseFloat(inputStr.replace(',', '.')) * 100` gerundet → `paidCents`
   - Bei ungültigem Input (NaN, leer) → paidCents = 0

5. **Live-Ergebnis** (erscheint sobald paidCents >= totalCents):
   - Differenz = paidCents - totalCents
   - Zwei Schnellauswahl-Buttons: "Kein Wechselgeld" (changeCents = 0) und "Alles zurück" (changeCents = Differenz)
   - State `changeCents: number` (default 0)
   - Zeile: "Wechselgeld" + Betrag
   - Grüne Box (bg-emerald-50 rounded-xl p-3): "Spende" + `formatEur(Math.max(0, differenz - changeCents))`

6. **Fehlerhinweis** (wenn paidCents > 0 && paidCents < totalCents):
   - `bg-rose-50 text-rose-600 rounded-xl p-3 text-sm` — "Betrag zu gering. Mindestens X nötig."

7. **Abschließen-Button** — `h-14 w-full rounded-xl text-xl font-semibold bg-sky-400 text-slate-800 active:bg-sky-500` — disabled wenn paidCents < totalCents
   - `onPointerDown={() => onComplete(paidCents, changeCents)}`

Alle Buttons verwenden `onPointerDown` (nicht onClick — Touch-Optimierung für iPad).
Min-Tap-Target 44px für alle interaktiven Elemente.

Bestehenden Import von `NumPad` entfernen (wird nicht mehr als Vollseite verwendet). `formatEur` bleibt.
