In `server/src/routes/import.ts` nach `const buffer = await data.toBuffer();` eine Magic-Byte-Prüfung einfügen, bevor `parseSuedNordKontorPdf(buffer)` aufgerufen wird.

PDF Magic Bytes: Die ersten 4 Bytes eines gültigen PDFs sind `%PDF` (hex: `25 50 44 46`).

```typescript
// Nach: const buffer = await data.toBuffer();
// Vor: const rows = await parseSuedNordKontorPdf(buffer);

function isPdf(buf: Buffer): boolean {
  return buf.length >= 4 &&
    buf[0] === 0x25 && // %
    buf[1] === 0x50 && // P
    buf[2] === 0x44 && // D
    buf[3] === 0x46;   // F
}

if (!isPdf(buffer)) {
  return reply.status(400).send({ error: 'Datei ist kein gültiges PDF (Magic Bytes ungültig)' });
}
```

Die `isPdf`-Funktion als Modul-Level-Hilfsfunktion (außerhalb von `importRoutes`) definieren, damit sie ohne Fastify-Kontext testbar ist.

Gesamter Post-Handler danach:
```typescript
fastify.post('/import/parse', async (request, reply) => {
  const data = await request.file();
  if (!data) {
    return reply.status(400).send({ error: 'Keine Datei uebermittelt' });
  }
  if (!data.filename.toLowerCase().endsWith('.pdf')) {
    return reply.status(400).send({ error: 'Nur PDF-Dateien erlaubt' });
  }

  try {
    const buffer = await data.toBuffer();
    if (!isPdf(buffer)) {
      return reply.status(400).send({ error: 'Datei ist kein gültiges PDF (Magic Bytes ungültig)' });
    }
    const rows = await parseSuedNordKontorPdf(buffer);
    return reply.send({ rows, filename: data.filename });
  } catch (err) {
    fastify.log.error(err, 'PDF-Parsing fehlgeschlagen');
    return reply.status(422).send({
      error: 'PDF konnte nicht verarbeitet werden',
      detail: err instanceof Error ? err.message : String(err),
    });
  }
});
```

In `server/src/lib/pdfParser.ts` die exportierte `parseSuedNordKontorPdf(buffer: Buffer)` Funktion in einen Promise.race()-Wrapper einschließen.

Strategie: Die eigentliche Parsing-Logik bleibt unverändert in einer internen `_parsePdf(buffer)` Funktion. Die exportierte `parseSuedNordKontorPdf` ist der öffentliche Wrapper mit Timeout.

```typescript
// Die bestehende Implementierung von parseSuedNordKontorPdf umbenennen zu:
async function _parsePdf(buffer: Buffer): Promise<ParsedInvoiceRow[]> {
  // ... alle bestehenden Zeilen der Funktion unverändert ...
}

// Neuer Export:
export async function parseSuedNordKontorPdf(buffer: Buffer): Promise<ParsedInvoiceRow[]> {
  const PDF_PARSE_TIMEOUT_MS = 30_000;

  const timeoutPromise = new Promise<never>((_, reject) =>
    setTimeout(
      () => reject(new Error(`PDF-Parsing Timeout: Verarbeitung dauerte länger als ${PDF_PARSE_TIMEOUT_MS / 1000}s`)),
      PDF_PARSE_TIMEOUT_MS,
    )
  );

  return Promise.race([_parsePdf(buffer), timeoutPromise]);
}
```

Wichtig: Die Timeout-Konstante `PDF_PARSE_TIMEOUT_MS` als benannte Konstante definieren (nicht magische Zahl). Das `export` Keyword vom alten `parseSuedNordKontorPdf` entfernen und auf die neue Wrapper-Funktion verschieben.
