<task type="checkpoint:human-action" gate="blocking">
  <name>Task 2: Apache Custom Config fuer Traefik-Proxy eintragen</name>
  <action>
Der User muss die Apache Custom Config in KeyHelp eintragen. Laut CLAUDE.md darf die Config NICHT direkt auf dem Server geaendert werden — immer ueber KeyHelp Web-UI.

Die Custom Config fuer den HTTPS-vHost von fairstand.godsapp.de:

```apache
ProxyPass / http://127.0.0.1:8888/
ProxyPassReverse / http://127.0.0.1:8888/
ProxyPreserveHost On
RequestHeader set X-Real-IP %{REMOTE_ADDR}s
```

Diese Config leitet alle Requests an Traefik (Port 8888) weiter. Traefik routet dann anhand der Docker-Labels (Host-Header) zu den richtigen Containern.

Websocket-Support wird hier NICHT benoetigt (kein WebSocket im Fairstand).

In KeyHelp: Domain waehlen → vHost-Einstellungen → Custom Config (HTTPS) → Config einfuegen → Speichern.

Die Config-Datei wird dann automatisch geschrieben nach:
`/etc/apache2/keyhelp/custom_vhosts/revisor_fairstand.godsapp.de_https.conf`
  </action>
  <verify>
    <automated>ssh root@server.godsapp.de "cat /etc/apache2/keyhelp/custom_vhosts/revisor_fairstand.godsapp.de_https.conf 2>/dev/null || echo 'Config noch nicht vorhanden'" 2>/dev/null || echo "SSH nicht verfuegbar — User muss manuell pruefen"</automated>
  </verify>
  <acceptance_criteria>
- Die Datei `/etc/apache2/keyhelp/custom_vhosts/revisor_fairstand.godsapp.de_https.conf` existiert auf dem Server
- Sie enthaelt `ProxyPass / http://127.0.0.1:8888/`
- Sie enthaelt `ProxyPreserveHost On`
- Apache laeuft ohne Fehler: `apachectl configtest` auf dem Server gibt `Syntax OK`
  </acceptance_criteria>
  <resume-signal>Tippe "config-ready" wenn die Apache Custom Config in KeyHelp eingetragen ist</resume-signal>
  <done>Apache leitet fairstand.godsapp.de korrekt an Traefik:8888 weiter</done>
</task>
