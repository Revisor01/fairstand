**Endpoint A: GET /reports/sales-csv** (EXP-01)
- Session: `const session = (request as any).session as { shopId: string }; const shopId = session.shopId;`
- Query-Parameter: `from` und `to` (Unix-ms als Strings) — bei Fehlen: 400 zurückgeben
- SQL: Alle Sales für shopId zwischen fromTs und toTs WHERE cancelled_at IS NULL AND (type IS NULL OR type = 'sale'), geordnet nach created_at
- SELECT: id, items (jsonb), total_cents, paid_cents, change_cents, donation_cents, created_at
- CSV-Zeilen: Pro Sale UND pro Item eine Zeile (flatMap), Spalten:
  `['Datum', 'Uhrzeit', 'Artikel', 'Menge', 'VK (EUR)', 'EK (EUR)', 'Summe Artikel (EUR)', 'Gesamtsumme (EUR)', 'Bezahlt (EUR)', 'Wechselgeld (EUR)', 'Spende (EUR)']`
  - Datum: `new Date(sale.created_at).toLocaleDateString('de-DE')`
  - Uhrzeit: `new Date(sale.created_at).toLocaleTimeString('de-DE', { hour: '2-digit', minute: '2-digit' })`
  - VK (EUR): `(item.salePrice / 100).toFixed(2)` — Komma NICHT ersetzen (CSV-Stringify mit quoted:true stellt sicher, dass Dezimalpunkt korrekt ist)
  - EK (EUR): `((item.purchasePrice ?? 0) / 100).toFixed(2)`
  - Summe Artikel: `(item.salePrice * item.quantity / 100).toFixed(2)`
  - Gesamtsumme/Bezahlt/Wechselgeld/Spende: nur in erster Item-Zeile des Sales, sonst leer
- Headers: `Content-Type: text/csv; charset=utf-8`, `Content-Disposition: attachment; filename="verkaufshistorie-{YYYY-MM-DD}.csv"`
- csv-stringify Optionen: `{ delimiter: ';', bom: true, quoted: true }`
- Error-Handling: `stringifier.on('error', ...)` → 500
- Streaming: `reply.send(stringifier)` NACH `stringifier.end()`

**Endpoint B: GET /reports/inventory-csv** (EXP-02)
- Session + shopId wie gehabt
- Query-Parameter: `year` (String) — bei Fehlen: 400
- Inventur-Daten: IDENTISCHE SQL-Queries wie im bestehenden GET /reports/inventory (yearStart, yearEnd, inventoryResult, stockValueResult) — Code DUPLIZIEREN nicht extrahieren (bleibt lesbar)
- CSV-Spalten: `['Artikelname', 'Artikelnummer', 'Bestand', 'Verkauft', 'VK-Umsatz (EUR)', 'EK-Kosten (EUR)', 'Bestandswert (EUR)']`
- Pro Item: name, article_number, current_stock, sold_qty, revenue_cents/100 toFixed(2), cost_cents/100 toFixed(2), (current_stock * current_ek_cents)/100 toFixed(2)
  - current_ek_cents: `ek_breakdown[0]?.ek_cents ?? 0` (aktueller EK = letzter bekannter EK-Preis aus ek_breakdown)
- Letzte Zeile (Summzeile): `['', '', '', '', '', 'GESAMT:', (total_stock_value_cents / 100).toFixed(2)]`
- Dateiname: `inventur-{year}-{shopId}.csv`
- Gleiche csv-stringify Optionen + Error-Handling wie Endpoint A
