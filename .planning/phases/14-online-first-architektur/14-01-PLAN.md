In `client/src/sync/engine.ts` die Funktion `downloadProducts()` (Zeilen 81-112) ersetzen.

**KRITISCH — Pitfall vermeiden:** `fetch()` MUSS vor der Dexie-Transaktion abgeschlossen sein. IDB-Transaktionen time-outen wenn async I/O innerhalb der Transaktion wartet.

**KRITISCH — shopId-Filter:** NIEMALS `db.products.clear()` verwenden. Stattdessen `db.products.where('shopId').equals(shopId).delete()` — sonst werden Produkte anderer Shops gelöscht (Multi-Laden-Architektur).

**Neuer Code für downloadProducts():**

```typescript
export async function downloadProducts(): Promise<number> {
  const shopId = getShopId();
  const res = await fetch(`/api/products?shopId=${shopId}`);
  if (!res.ok) throw new Error(`Download fehlgeschlagen: ${res.status}`);
  const serverProducts: ServerProduct[] = await res.json();

  const mapped: Product[] = serverProducts.map(sp => ({
    id: sp.id,
    shopId: sp.shopId,
    articleNumber: sp.articleNumber,
    name: sp.name,
    category: sp.category,
    purchasePrice: sp.purchasePrice,
    salePrice: sp.salePrice,
    vatRate: sp.vatRate,
    stock: sp.stock,
    minStock: sp.minStock,
    active: Boolean(sp.active),
    imageUrl: sp.imageUrl ?? undefined,
    updatedAt: sp.updatedAt,
  }));

  await db.transaction('rw', db.products, async () => {
    await db.products.where('shopId').equals(shopId).delete();
    await db.products.bulkPut(mapped);
  });

  return mapped.length;
}
```

- Die Variable `let upserted = 0;`

- Das `return upserted;` am Ende

**Rückgabewert ändert sich:** War `upserted` (Anzahl geänderter), ist jetzt `mapped.length` (Gesamtanzahl). Das ist korrekt — ProductList.handleDownloadSync zeigt "X Produkte aktualisiert", was auch für Server-Replace passt.
